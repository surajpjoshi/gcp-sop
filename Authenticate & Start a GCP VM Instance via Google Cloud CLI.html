Standard Operating Procedure (SOP)

Authenticate & Start a GCP VM Instance via Google Cloud CLI

Objective

To log in to Google Cloud CLI, set the correct project, and start a virtual machine (VM) instance.

Pre-requisites

  • Google Cloud CLI installed
  • Internet connectivity
  • Permissions to access and start GCP VM

Procedure

Step 1: Open Google Cloud CLI

C:\Users\<username>\AppData\Local\Google\Cloud SDK>

Step 2: Authenticate to Google Cloud

gcloud auth login

This opens a browser. Log in with your Google account.

Step 3: List Available Projects

gcloud projects list
PROJECT_ID NAME PROJECT_NUMBER
evevg-464605 evevg 239527817814
fiery-chess-464605-d1 My First Project 919329399059
test-464817 test 895226267251

Step 4: Set Active Project

gcloud config set project test-464817

Step 5: List VM Instances

gcloud compute instances list

Step 6: Start the VM

gcloud compute instances start eveng --zone=us-central1-f

Step 7: Confirm VM is Running

gcloud compute instances list

Troubleshooting

Issue Resolution
Permission Error Check IAM roles and project selection
Command Not Found Use gcloud prefix for all commands
No External IP Assign static or ephemeral IP to the VM
