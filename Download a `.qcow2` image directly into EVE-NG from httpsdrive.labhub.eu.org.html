SOP: Download and Deploy `.qcow2` Image in EVE-NG QEMU Folder

Purpose

To download a `.qcow2` image directly into EVE-NG and prepare it for use in virtual labs.

Prerequisites

- EVE-NG is installed and running on your Linux VM.
- You have root or sudo access.
- Internet access from EVE-NG VM.

Steps

Step 1: SSH into your EVE-NG machine
ssh root@<eve-ng-ip>
# OR if on GCP:
gcloud compute ssh eveng --zone=us-central1-f

Step 2: Navigate to the QEMU folder
cd /opt/unetlab/addons/qemu/

Step 3: Create a new directory for the image
mkdir win10-vl19
cd win10-vl19

Step 4: Download the `.qcow2` image using `wget`
wget "https://drive.labhub.eu.org/0:/addons/qemu/win-10-x64-VL19/virtioa.qcow2" -O virtioa.qcow2

# If SSL issues occur, use:
wget --no-check-certificate "https://drive.labhub.eu.org/0:/addons/qemu/win-10-x64-VL19/virtioa.qcow2" -O virtioa.qcow2

Step 5: Rename the downloaded file (if required)
mv virtioa.qcow2 hda.qcow2

Step 6: Fix EVE-NG permissions
/opt/unetlab/wrappers/unl_wrapper -a fixpermissions

Outcome

The image is downloaded to the correct folder and is ready to be used in EVE-NG topology.

Example Folder Structure


/opt/unetlab/addons/qemu/win10-vl19/
 └── hda.qcow2
